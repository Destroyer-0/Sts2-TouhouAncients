| 类型 | 基类 | 命名空间 | 说明 |
|------|------|---------|------|

### 2.4 添加新先古之民注意事项
每次添加**新的 Ancient 角色**时，必须在 `scripts/TouhouAncientsConfig.cs` 中：
1. **禁用节 (`BanAncients`)** — 添加一个 `public static bool Ban{角色名} { get; set; } = false;`
2. **强制节 (`ForcedAncients`)** — 添加一个 `public static bool Force{角色名} { get; set; } = false;`
3. 在 `localization/zhs/settings_ui.json` 中添加对应的标题本地化键
4. 在 Ancient 类中重写 `IsValidForAct`（检查 Ban）和 `ShouldForceSpawn`（返回 Force）

### 4.3 Power 应用

```csharp
// 给自己加永久 Buff
await PowerCmd.Apply<DexterityPower>(自己Creature, 1m, null, null);

// 给自己加临时 Buff (下回合移除)
await PowerCmd.Apply<EnergyNextTurnPower>(自己Creature, 1m, applier, null);

// 给敌人加 Debuff
await PowerCmd.Apply<FrailPower>(敌人, amount, applier, null);

// 给所有敌人
var enemies = 自己Creature.CombatState.GetOpponentsOf(自己Creature).Where(c => c.IsAlive);
await PowerCmd.Apply<StrengthPower>(enemies, amount, null, null);

// 使用 canonical power 实例而非泛型 (用于可变类型)
await PowerCmd.Apply(power, enemy, amount, applier, null);
```

### 4.8 牌组附魔选择模式（选牌→附魔→VFX）

```csharp
public override async Task AfterObtained()
{
    var player = base.Owner;
    var enchantment = ModelDb.Enchantment<SomeEnchantment>();

    // 从牌组选牌附魔
    var selected = (await CardSelectCmd.FromDeckForEnchantment(
        player,
        enchantment,
        1,
        new CardSelectorPrefs(CardSelectorPrefs.EnchantSelectionPrompt, 1)
    )).FirstOrDefault();

    if (selected != null)
    {
        CardCmd.Enchant<SomeEnchantment>(selected, 1m);
        var vfx = NCardEnchantVfx.Create(selected);
        if (vfx != null)
            NRun.Instance?.GlobalUi.CardPreviewContainer.AddChildSafely(vfx);
    }
}
```
参考 `MoriyaGohei`, `BloodFang`.

    // result.UnblockedDamage = 实际穿透伤害

### 4.10 动态变量 (DynamicVar) 运行时修改

某些遗物的描述需要运行时动态值（如按当前最大生命计算），可以在 `AfterObtained` 中修改：

```csharp
protected override IEnumerable<DynamicVar> CanonicalVars => [new DynamicVar("LoseHp", 0)];

public override async Task AfterObtained()
{
    // 运行时计算并更新动态变量值
    base.DynamicVars["LoseHp"].BaseValue = Math.Floor(player.Creature.MaxHp / 2);
}
```

常用 DynamicVar 子类：

| 类名 | 默认Key | 用途 |
|------|---------|------|
| `DynamicVar` | 自定义 | 通用数值 |
| `MaxHpVar` | `MaxHp` | 最大生命值 |
| `HpLossVar` | `HpLoss` | 损失生命值 |
| `EnergyVar` | `Energy` | 能量 |
| `CardsVar` | `Cards` | 卡牌数量 |
| `DamageVar` | `Damage` | 伤害 |
| `StringVar` | 自定义 | 字符串（如附魔名） |

### 4.11 卡牌开发模式

#### 完整卡牌模板

```csharp
using MegaCrit.Sts2.Core.Entities.Cards;
using MegaCrit.Sts2.Core.GameActions.Multiplayer;
using MegaCrit.Sts2.Core.HoverTips;
using MegaCrit.Sts2.Core.Localization.DynamicVars;
using MegaCrit.Sts2.Core.Models.CardPools;

namespace TouhouAncients.Scripts.cards;

[Pool(typeof(EventCardPool))]
public class ExampleCard : TouhouAncientCards
{
    private const int energyCost = 1;          // 基础费用
    private const CardType type = CardType.Skill;  // Skill / Attack / Power / Curse
    private const CardRarity rarity = CardRarity.Ancient; // 自定义稀有度
    private const TargetType targetType = TargetType.Self; // 目标类型
    private const bool shouldShowInCardLibrary = true;

    // 动态变量：用于描述文本中的 {Key} 占位符
    protected override IEnumerable<DynamicVar> CanonicalVars =>
    [
        new DynamicVar("ScryAmount", 3),
        new EnergyVar(1)  // 升级时减少费用
    ];

    // 悬停提示：关键字说明 + 能量费用提示
    protected override IEnumerable<IHoverTip> ExtraHoverTips =>
    [
        HoverTipFactory.FromKeyword(TouhouAncientKeywords.ExampleKeyword),
        base.EnergyHoverTip
    ];

    // 卡牌关键字（固有、保留、虚无等）
    public override IEnumerable<CardKeyword> CanonicalKeywords => [CardKeyword.Retain];

    // 无参构造函数
    public ExampleCard() : base(energyCost, type, rarity, targetType, shouldShowInCardLibrary)
    {
    }

    // 打出时效果
    protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
    {
        // 实现卡牌效果
        // await SomeCmd.DoSomething(...);
        
        // 临时费用修改（配合 EnergyVar 使用）
        EnergyCost.AddThisTurn(DynamicVars.Energy.IntValue);
    }

    // 自定义弃牌去向（可选）
    protected override PileType GetResultPileType()
    {
        PileType result = base.GetResultPileType();
        if (result != PileType.Discard) return result;
        return PileType.Hand; // 改为回手
    }

    // 升级时数值变化（可选）
    protected override void OnUpgrade()
    {
        base.DynamicVars.Energy.UpgradeValueBy(-1m); // 费用-1
    }
}
```

#### 卡牌关键虚方法

| 方法 | 用途 | 说明 |
|------|------|------|
| `OnPlay` | 打出效果 | 卡牌打出时的核心逻辑 |
| `GetResultPileType` | 弃牌去向 | 返回 `PileType.Hand` 可回手 |
| `OnUpgrade` | 升级变化 | 修改 `DynamicVars` 改变数值 |
| `CanPlay` | 能否打出 | 限制打出的条件（可选） |
| `BeforeFlush` | 费用修改 | 手牌检测阶段调整费用 |

## 6. 常见注意事项

### 6.1 多人模式
- 每个 Hook 都要检查 `player != base.Owner`
- 参考 `Yishixingqile`, `Yiyandingzhen` 的实现

### 6.2 存档序列化
- `[SavedProperty]` 标记的属性会自动持久化
- Mod 中的类型需在 `Entry.cs` 注册: `SavedPropertiesTypeCache.InjectTypeIntoCache<Type>()`

### 6.3 脚本注册
- `abstract` 类不会被 Godot 脚本扫描器注册
- `TouhouAncientRelics` 是 abstract，不会生成遗物实例
- `TouhouAncientCards` 是 abstract，不会生成卡牌实例

### 6.4 卡牌注册说明
- 卡牌无需在 `Entry.cs` 中手动注册，`ScriptManagerBridge.LookupScriptsInAssembly` 会自动扫描
- `TouhouAncientCards` 必须标记为 `abstract`，否则会因无参构造函数缺失而报错
  `TouhouAncientCards No parameterless constructor defined`
- 具体卡牌类必须有 `public` 无参构造函数调用 `base(energyCost, type, rarity, targetType, shouldShowInCardLibrary)`
- 卡牌使用 `[Pool(typeof(EventCardPool))]` 属性指定出现的池子（如事件、商店等）

### 6.4 附魔的 Hook 生命周期
- 附魔 (`CustomEnchantmentModel`) 继承自 `AbstractModel`
- 战斗中卡牌的附魔会被 `CombatState.IterateHookListeners()` 包含 → **可以接收全局 Hook**
- 可用但**附魔独有的** Hook（对应 `CardModel` 的虚方法）：
  - `OnPlay` — 打出时
  - `AfterCardPlayed` — 打出后
  - `BeforeFlush` — 手牌检测阶段（可用于费用修改）
  - `AfterCardChangedPiles` — 卡牌换牌堆时
  - `BeforeCardRemoved` — 卡牌被移除时
- 可用的**全局** Hook（通过 `AbstractModel` 继承）：
  - `AfterDamageGiven` — 造成伤害后
  - `BeforeDamageReceived` — 受到伤害前
- 参考 `Bloodshed`, `SnakeBiteUpgrade`, `HighQuality`

### 6.5 `[SavedProperty]` 注册规则
- `[SavedProperty]` 标记的属性会自动持久化到存档
- Mod 中的类型需在 `Entry.cs` 的 `Init()` 中注册: `SavedPropertiesTypeCache.InjectTypeIntoCache(typeof(ClassName))`
- 适用于**遗物**和**附魔**的运行时状态持久化（如 `BrainInAVat._blockedCardIds`）
- 注册位置必须是全类型引用，不区分 namespace：
  ```csharp
  SavedPropertiesTypeCache.InjectTypeIntoCache(typeof(HighQuality));
  SavedPropertiesTypeCache.InjectTypeIntoCache(typeof(BrainInAVat));
  ```
- 参考：`HighQuality`（附魔费用减量）、`BrainInAVat`（遗物屏蔽卡牌黑名单），`Miracle` / `SnakeBiteUpgrade` / `Bloodshed` / `Recollection`（无 `[SavedProperty]` → 不注册）

### 6.6 一次性附魔效果：使用 EnchantmentStatus.Disabled

对于只需触发一次的附魔效果（如回忆：首次弃牌自动打出），无需 `_hasTriggered` 私有字段标记。

```csharp
// ❌ 不要这样
private bool _hasTriggered;
// ... if (_hasTriggered) return; _hasTriggered = true;

// ✅ 推荐
base.Status = EnchantmentStatus.Disabled;
```

`EnchantmentStatus.Disabled` 会将附魔标记为停用状态，效果等同于"已消耗"：
- 附魔不再出现在 UI 悬浮提示中
- 后续 `AfterCardChangedPiles`、`OnPlay` 等 Hook **不会再被调用**（因为 `Hook.cs` 的 `IterateHookListeners` 遍历时不会回调）
- 不需要额外字段，读档后也自动保留状态

**注意**: `base.Status` 需要引入 `MegaCrit.Sts2.Core.Entities.Enchantments` 命名空间。
参考 `Recollection` 附魔实现。

```csharp
// 1. 在 relics.json 中定义关键词
// "TOUHOUANCIENTS-FILTER_SNAKE_KEYWORDS.title": "蛇,snake,Snake"

// 2. 加载关键词（Lazy 单次加载，支持多语言）
private static readonly Lazy<string[]> _keywords = new(() =>
    new LocString("relics", "TOUHOUANCIENTS-FILTER_XXX_KEYWORDS.title")
        .GetFormattedText()
        .Split(',', StringSplitOptions.TrimEntries | StringSplitOptions.RemoveEmptyEntries));

// 3. 匹配卡牌标题（大小写不敏感，只触发一次，非遍历 Tags）
private static bool IsMatch(CardModel card) =>
    _keywords.Value.Any(k => card.TitleLocString.GetFormattedText()
        .Contains(k, StringComparison.OrdinalIgnoreCase));
```

**攻击 Builder 模式（DamageCmd.Attack）**

推荐使用 builder 链式调用替代 `CreatureCmd.Damage`，以支持自定义特效和音效：
```csharp
await DamageCmd.Attack(base.DynamicVars.Damage.BaseValue)
    .FromCard(this)                          // 指定来源卡牌
    .Targeting(cardPlay.Target)              // 指定目标
    .WithHitFx("vfx/vfx_attack_blunt", null, "blunt_attack.mp3")  // VFX + SFX
    .Execute(choiceContext);                 // 执行
```
- `DamageCmd.Attack(decimal)` — 传入基础伤害数值或 `CalculatedDamageVar`
- `.FromCard(CardModel)` — 设置伤害来源（影响力量/敏捷加成）
- `.Targeting(Creature)` — 指定目标
- `.WithHitFx(string vfxPath, string? hitSparkPath, string? sfxPath)` — 自定义特效
- `.Execute(choiceContext)` — 执行攻击

需要引入 `MegaCrit.Sts2.Core.Commands`。
参考 `Mangle`（原版）、`DreamSeal`（Mod）。
