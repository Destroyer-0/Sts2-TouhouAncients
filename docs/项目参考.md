## 2. 命名规范

### 2.1 文件、类、命名空间

### 2.4 添加新先古之民注意事项
每次添加**新的 Ancient 角色**时，必须在 `scripts/TouhouAncientsConfig.cs` 中：
1. **禁用节 (`BanAncients`)** — 添加一个 `public static bool Ban{角色名} { get; set; } = false;`
2. **强制节 (`ForcedAncients`)** — 添加一个 `public static bool Force{角色名} { get; set; } = false;`
3. 在 `localization/zhs/settings_ui.json` 中添加对应的标题本地化键
4. 在 Ancient 类中重写 `IsValidForAct`（检查 Ban）和 `ShouldForceSpawn`（返回 Force）

**卡牌描述中的参数绑定：**
- 使用 `{DynamicVarName}` 自动替换为 `CanonicalVars` 中对应变量的运行时值

### 2.5 附魔属性规范

```csharp
public class Xxx : CustomEnchantmentModel  // 来自 BaseLib.Abstracts
{
    // 覆盖 CanEnchantCardType → 限制可附魔卡牌类型
    // 覆盖 CanEnchant(CardModel card) → 更精细的限制（如排除X费牌）
    // 覆盖 OnEnchant → 应用关键字/属性修改
    // 覆盖 OnPlay → 打出时效果 (可选)
}
```

`CanEnchant(CardModel card)` 在 `CanEnchantCardType` 之后调用，用于检查完整卡牌对象：

```csharp
public override bool CanEnchant(CardModel card)
{
    // 排除能量费用为X的牌（CostsX = 能量X费）
    if (card.EnergyCost.CostsX) return false;
    // 排除星数费用为X的牌（HasStarCostX = 星数X费）
    if (card.HasStarCostX) return false;
    return true;
}
```
> 判断 X 费牌的两个属性：`card.EnergyCost.CostsX`（能量X费）和 `card.HasStarCostX`（星数X费）。

### 2.7 卡牌开发规范

#### 卡牌基类约定
- 所有 Mod 卡牌继承 `TouhouAncientCards`（`scripts/cards/TouhouAncientCards.cs`），该类继承 `CustomCardModel`
- `TouhouAncientCards` 标记为 `abstract`，避免被 `ScriptManagerBridge` 直接实例化
- 基类已提供无参构造器（`protected`），满足 Godot 反射扫描需求；子类无需额外处理
- 肖像路径自动解析：`res://sprite/cards/{类名}.png`
- 命名空间一律：`TouhouAncients.Scripts.cards`
- `[Pool(typeof(EventCardPool))]` → 卡牌出现池子；不加则不会出现在任何奖励中

#### 构造函数规范
- 必须提供**无参构造函数**：`public Xxx() : base(energyCost, type, rarity, targetType, shouldShowInCardLibrary)`
- `base()` 的参数在类顶部定义为 `private const`，方便查阅
- 示例：`SatoriEye.cs` — `0`费技能，`CardRarity.Ancient`，目标自身
- `shouldShowInCardLibrary` 控制是否在图鉴中显示

#### 无参构造器说明
`TouhouAncientCards` 标记为 `abstract` → `ScriptManagerBridge` 会跳过抽象基类，不会尝试实例化。
但**具体卡牌子类**（如 `SatoriEye`）**必须**提供 `public` 无参构造器，链向基类带参：
```csharp
public SatoriEye() : base(energyCost, type, rarity, targetType, shouldShowInCardLibrary) { }
```
这是为了：
1. `ScriptManagerBridge.LookupScriptsInAssembly` 通过 `Activator.CreateInstance` 扫描程序集时实例化具体卡牌类型
2. Godot 序列化/反序列化也需要无参构造器

> 基类 `abstract` → 不会被扫描；子类 `public 无参` → 被扫描 + 链入基类带参构造器

#### CanonicalVars 运行时变量
| 变量类型 | 用途 | 示例 |
|---------|------|------|
| `DynamicVar` | 通用数值变量 | `new DynamicVar("Damage", 8)` |
| `EnergyVar` | 能量相关 | `new EnergyVar(2)` → 动态能量增减 |
| `BlockVar` | 格挡相关 | `new BlockVar(5)` |
| `CardsVar` | 卡牌数量 | `new CardsVar(3)` |

- 变量通过 `base.DynamicVars["Key"].BaseValue` 读写
- 升级用 `base.DynamicVars["Key"].UpgradeValueBy(value)` 而非修改字段

#### 卡牌 Hook 汇总

| Hook | 说明 | 示例 |
|------|------|------|
| `OnPlay` | 打出时效果 | `SatoriEye` — 预见5 + 本回合耗能+2 |
| `OnUpgrade` | 升级时数值变化 | `SatoriEye` — 能量消耗减少1 |
| `GetResultPileType` | 自定义弃牌去向 | `SatoriEye` — 返回手牌而非弃牌堆 |
| `CanonicalKeywords` | 卡牌关键字 | `SatoriEye` — `[CardKeyword.Retain]` |
| `ExtraHoverTips` | 额外悬停提示 | 自定义关键字或 Power 说明 |
| `ShouldTakeExtraTurn` | 额外回合判定 | `ServantSakuya`（旧版） |
| `AfterTakingExtraTurn` | 额外回合开始前 | 同上 |

#### 本地化键名
- 卡牌标题：`TOUHOUANCIENTS-{全大写蛇形}.title`（如 `SatoriEye` → `SATORI_EYE`）
- 卡牌描述：`{KEY}.description`
- 升级后描述：`{KEY}.descriptionUpgraded`（可选，不提供则自动追加 `+`）
- 参数绑定：描述中使用 `{DynamicVarName}` 自动替换运行时数值

#### 注册说明
- 卡牌**无需**在 `Entry.cs` 中手动注册（区别于 `[SavedProperty]` 类型）
- `[Pool(typeof(EventCardPool))]` 属性自定池子
- `ScriptManagerBridge.LookupScriptsInAssembly` 自动发现所有卡牌类型

### 4.10 遗物：卡牌奖励额外添加（TryModifyCardRewardOptions）

参照 `LastingCandy` 模式，在现有卡牌奖励基础上额外添加自定义卡牌：

```csharp
public override bool TryModifyCardRewardOptions(Player player, List<CardCreationResult> options, CardCreationOptions creationOptions)
{
    if (base.Owner != player) return false;

    var customCards = new List<CardModel>
    {
        ModelDb.Card<SomeCard>()
    };

    var servantOptions = new CardCreationOptions(customCards, CardCreationSource.Other, CardRarityOddsType.Uniform)
        .WithFlags(CardCreationFlags.NoModifyHooks | CardCreationFlags.NoCardPoolModifications);

    var created = CardFactory.CreateForReward(base.Owner, 1, servantOptions).FirstOrDefault();
    if (created?.Card == null) return false;

    var result = new CardCreationResult(created.Card);
    result.ModifyCard(created.Card, this);
    options.Add(result);
    return true;
}
```
参考 `NightServant`、`LastingCandy`。

### 4.11 遗物：阻止特定卡牌加入牌组（ShouldAddToDeck）

public override bool ShouldAddToDeck(CardModel card)

    return true;
}
```

## 6. 常见注意事项

### 6.1 多人模式
- 每个 Hook 都要检查 `player != base.Owner`
- 参考 `Yishixingqile`, `Yiyandingzhen` 的实现

### 6.2 存档序列化
- `[SavedProperty]` 标记的**属性**会自动持久化
- **必须使用自动属性（auto-property）**，不能标记在字段上：
  ```csharp
  // ✅ 正确：属性
  [SavedProperty] private string TouhouAncients_MyData { get; set; }
  
  // ❌ 错误：字段（BaseLib 会忽略字段上的 [SavedProperty]）
  [SavedProperty] private string _myData;
  ```
- 建议使用 `TouhouAncients_` 前缀避免与其他 Mod 的 SavedProperty 冲突
- Mod 中的类型需在 `Entry.cs` 注册: `SavedPropertiesTypeCache.InjectTypeIntoCache<Type>()`

### 6.3 脚本注册
- `abstract` 类不会被 Godot 脚本扫描器注册
- `TouhouAncientRelics` 是 abstract，不会生成遗物实例
- `TouhouAncientCards` 是 abstract，不会生成卡牌实例

### 6.4 卡牌注册说明
- 卡牌无需在 `Entry.cs` 中手动注册，`ScriptManagerBridge.LookupScriptsInAssembly` 会自动扫描
- `TouhouAncientCards` 必须标记为 `abstract`，否则会因无参构造函数缺失而报错
  `TouhouAncientCards No parameterless constructor defined`
- 具体卡牌类必须有 `public` 无参构造函数调用 `base(energyCost, type, rarity, targetType, shouldShowInCardLibrary)`
- 卡牌使用 `[Pool(typeof(EventCardPool))]` 属性指定出现的池子（如事件、商店等）

### 6.4 附魔的 Hook 生命周期
- 附魔 (`CustomEnchantmentModel`) 继承自 `AbstractModel`
- 战斗中卡牌的附魔会被 `CombatState.IterateHookListeners()` 包含 → **可以接收全局 Hook**
- 可用但**附魔独有的** Hook（对应 `CardModel` 的虚方法）：
  - `OnPlay` — 打出时
  - `AfterCardPlayed` — 打出后
  - `BeforeFlush` — 手牌检测阶段（可用于费用修改）
  - `AfterCardChangedPiles` — 卡牌换牌堆时
  - `BeforeCardRemoved` — 卡牌被移除时
- 可用的**全局** Hook（通过 `AbstractModel` 继承）：
  - `AfterDamageGiven` — 造成伤害后
  - `BeforeDamageReceived` — 受到伤害前
- 参考 `Bloodshed`, `SnakeBiteUpgrade`, `HighQuality`

### 6.5 `[SavedProperty]` 注册规则
- `[SavedProperty]` 标记的**属性**会自动持久化到存档
- **必须使用自动属性（auto-property）**，不能标记在字段上：`[SavedProperty] private string Name { get; set; }`
- 建议的命名格式：`TouhouAncients_{描述}`，避免与其他 Mod 冲突（BaseLib 会 warn 无前缀的 SavedProperty）
- 参考 `HighQuality.cs`（附魔费用减量）：
  ```csharp
  [SavedProperty] private int TouhouAncients_EnergyDelta { get; set; }
  ```
- 参考 `Tribute.cs`（卡牌进度持久化）：
  ```csharp
  [SavedProperty] private int SavedGoldContributed { get; set; }
  ```
- Mod 中的类型需在 `Entry.cs` 的 `Init()` 中注册：
  ```csharp
  SavedPropertiesTypeCache.InjectTypeIntoCache(typeof(HighQuality));
  SavedPropertiesTypeCache.InjectTypeIntoCache(typeof(BrainInAVat));
  ```
  - **遗物**和**附魔**有 `[SavedProperty]` 才需注册
  - 仅 `[Pool]` 属性而无 `[SavedProperty]` 的类型（如 `Miracle` / `Bloodshed` / `Recollection`）→ 不注册
- 适用于**遗物**、**附魔**和**卡牌**的运行时状态持久化

### 6.6 一次性附魔效果：使用 EnchantmentStatus.Disabled

对于只需触发一次的附魔效果（如回忆：首次弃牌自动打出），无需 `_hasTriggered` 私有字段标记。

```csharp
// ❌ 不要这样
private bool _hasTriggered;
// ... if (_hasTriggered) return; _hasTriggered = true;

// ✅ 推荐
base.Status = EnchantmentStatus.Disabled;
```

`EnchantmentStatus.Disabled` 会将附魔标记为停用状态，效果等同于"已消耗"：
- 附魔不再出现在 UI 悬浮提示中
- 后续 `AfterCardChangedPiles`、`OnPlay` 等 Hook **不会再被调用**（因为 `Hook.cs` 的 `IterateHookListeners` 遍历时不会回调）
- 不需要额外字段，读档后也自动保留状态

**注意**: `base.Status` 需要引入 `MegaCrit.Sts2.Core.Entities.Enchantments` 命名空间。
参考 `Recollection` 附魔实现。

### 4.15 卡牌池创建 + 网格选择（参照 SeaGlass / CosmicDecree）

创建指定角色牌池、指定稀有度的卡牌，供玩家选择加入牌组：

```csharp
// 创建卡牌池选项
var cardPool = ModelDb.Character<Regent>().CardPool;
var options = CardCreationOptions.ForNonCombatWithUniformOdds(
    new[] { cardPool },          // 卡牌池
    _ => true)                   // 筛选条件（全部）
    .WithFlags(CardCreationFlags.NoRarityModification 
              | CardCreationFlags.NoCardPoolModifications
              | CardCreationFlags.NoUpgradeRoll);

// 生成卡牌
var creationResults = CardFactory.CreateForReward(player, cardCount, options).ToList();

// 升级
foreach (var result in creationResults)
    if (result.Card.IsUpgradable) CardCmd.Upgrade(result.Card);

// 网格多选（min=0 表示可以不选）
var selected = (await CardSelectCmd.FromSimpleGridForRewards(
    new BlockingPlayerChoiceContext(),
    creationResults,
    player,
    new CardSelectorPrefs(promptLocString, minSelect: 0, maxSelect: creationResults.Count)
)).ToList();

// 选定牌加入牌组
var addResults = new List<CardPileAddResult>();
foreach (var card in selected)
    addResults.Add(await CardPileCmd.Add(card, PileType.Deck));
CardCmd.PreviewCardPileAdd(addResults, 2f);
```

### 4.17 Power 动态追踪全场模式（参照 LuckyRabbitPower）

当 buff 需要在生效期间给**所有战斗牌堆**中的目标牌动态附加效果时，使用组合 Hook 多方位捕获：

```csharp
public class MyTrackingPower : PowerModel
{
    public override PowerType Type => PowerType.Buff;
    public override PowerStackType StackType => PowerStackType.Counter;

    // ① Power 被应用/递增时 → 扫描全场已有牌
    public override async Task AfterApplied(Creature? applier, CardModel? cardSource)
    {
        var combatState = base.Owner.Player?.PlayerCombatState;
        if (combatState == null) return;

        var allPiles = new[] { combatState.Hand.Cards, combatState.DrawPile.Cards, combatState.DiscardPile.Cards };
        foreach (var pile in allPiles)
            foreach (var card in pile)
                ApplyEffect(card);
    }

    // ② 抽牌时 → 捕获新入手的牌
    public override async Task AfterCardDrawn(PlayerChoiceContext ctx, CardModel card, bool fromHandDraw)
    {
        if (card.Owner != base.Owner.Player) return;
        ApplyEffect(card);
    }

    // ③ 牌堆间移动时 → 捕获从牌组/外部进入战斗的牌
    public override async Task AfterCardChangedPiles(CardModel card, PileType oldPile, AbstractModel? source)
    {
        if (card.Owner != base.Owner.Player) return;
        if (card.Pile == null || !card.Pile.IsCombatPile) return;
        bool fromOutside = oldPile is PileType.None or PileType.Deck;
        if (fromOutside)
            ApplyEffect(card);
    }

    // ④ 回合结束自动移除
    public override async Task BeforeTurnEnd(PlayerChoiceContext ctx, CombatSide side)
    {
        if (side == base.Owner.Side) await PowerCmd.Remove(this);
    }

    private void ApplyEffect(CardModel card) { /* 根据条件给 card 附加效果 */ }
}
```
